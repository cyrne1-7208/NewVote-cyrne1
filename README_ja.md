# NewVote

**[English README](README.md)**

Spigot サーバー上でプレイヤー投票を実施し、投票状況を Tab リストにリアルタイム表示するプラグインです。

## 概要
NewVote を使用すると、ゲーム内を離れずに投票セッションを開始・終了できます。
プレイヤーはコマンド 1 つで投票し、結果は即座に Tab リストへ反映されます。
投票終了時には、得票数の順位と集計結果が全プレイヤーに表示されます。

### 対象ユーザー
- イベントやミニゲームで投票を行う Minecraft サーバーの管理者
- Tab リストで投票状況をリアルタイムに確認したいプレイヤー

## 特徴
- Tab リストへのリアルタイム表示: 未投票は `×`、投票済みは `✓`
- 投票終了時に全プレイヤーへ順位付き集計結果を表示
- 投票先の開示: 誰が誰に投票したかを Tab リストに表示
- オンラインプレイヤーまたは固定リストに基づく `/v` のタブ補完
- `config.yml` の `List` による固定投票先リストの設定
- 特定プレイヤー向けの演出機能（Yanaaaaa 専用）
- スタックトレース出力を制御できるデバッグログ機能
- Bukkit/Spigot API のみを使用しており、NMS 依存なし

## 必要環境
- Java 8 以上
- Spigot または Paper 1.15.2 以上

## インストール
1. プラグイン jar をビルドします。
2. jar をサーバーの `plugins/` フォルダにコピーします。
3. サーバーを再起動します。

```bash
mvn -DskipTests package
```

生成物: `target/NewVote-v8.77778.1.jar`

## クイックスタート
1. jar を `plugins/` に配置してサーバーを起動します。
2. OP としてログインし、`/vs` で投票を開始します。
3. プレイヤーが `/v <投票先>` で投票します。
4. 再度 `/vs` を実行して投票を締め切り、結果を表示します。

期待結果:

```text
全プレイヤーに順位付きの集計結果が表示されます。
各行に順位、プレイヤー名、得票数が表示されます。
```

## 使い方
投票の開始・締切（OP のみ）:

```text
/vs
```

投票:

```text
/v <投票先>
```

投票先の開示（OP のみ）:

```text
/vget
```

### 投票フロー
1. `/vs` — 投票開始。全プレイヤーの Tab リストに `×` が表示される。
2. `/v <投票先>` — 投票。自分の Tab エントリが `✓` に変わる。
3. `/vs` — 投票締切。全プレイヤーに順位付き集計結果が表示される。
4. `/vget` — 誰が誰に投票したかを Tab リストに表示する。
5. `/vs` — Tab リストの投票先表示をクリアする。

### やなーもーど

```text
/yvote          — やなーもーど ON/OFF 切替（Yanaaaaa 専用）
/yvote <順位>   — 演出発動の順位閾値を変更（Yanaaaaa 専用）
```

### コマンド一覧
| コマンド | 権限 | 説明 |
|---|---|---|
| `/v <投票先>` | `newvote.v`（既定: 全員） | 投票 |
| `/vs` | `newvote.vs`（既定: OP） | 投票開始・締切・Tab リストクリア |
| `/vget` | `newvote.vs`（既定: OP） | 投票先を Tab リストに表示 |
| `/yvote [順位]` | — | Yanaaaaa 専用コマンド |

## 設定
`config.yml`:

```yaml
List: []

debug:
  enabled: false
  logStackTrace: true
```

| キー | 既定値 | 説明 |
|---|---|---|
| `List` | `[]` | 固定投票先リスト。空のときはオンラインプレイヤーが対象。 |
| `debug.enabled` | `false` | コマンド実行時の状態遷移ログを出力する。 |
| `debug.logStackTrace` | `true` | エラーログにスタックトレースを含める。 |

固定リストの設定例:

```yaml
List:
  - "対象1"
  - "対象2"
```

`List` に要素がある場合、オンラインプレイヤーに関わらず固定リストのみが投票先になります。

## テスト
自動テストスイートはありません。
ローカルの Paper サーバーで小人数の環境を用意し、各ステップ後に Tab リストの更新を目視で確認してください。

## トラブルシューティング
- 投票が開始されない: OP 権限を持っているか、プラグインがエラーなく起動しているか確認します。
- 投票先が見つからない: プレイヤー名またはリストの表記が完全一致しているか確認します。
- サウンドエラー: プラグインは自動的に代替の `Sound` 値にフォールバックします。
- Tab リストが更新されない: バニラではなく Spigot/Paper サーバーで実行されているか確認します。
